Everything you need

Professional component authoring that generates AI-friendly, production-ready output.

T

Token-First Design

All visual properties controlled through design tokens. Saddle enforces consistency and auto-suggests tokenization for raw values.

AI

AI-Readable Components

design.md frontmatter gives AI explicit metadata: when to use components, constraints, and best practices. Claude Code understands your library.

MCP

Claude Code Integration

Bidirectional MCP server. Claude Code reads schemas, updates tokens, creates variants. One-click setup from the dashboard.

HMR

Live Dev Server Preview

Embed your running localhost inside Saddle. Edit tokens, file writes to disk, your real app updates via HMR. No refresh needed.

DOM

Element Inspector

Chrome DevTools-style element tree showing your component's full DOM hierarchy with computed styles per element.

npm

npm Package Export

One-click build: conditional exports, generated CSS Modules, global tokens.css, DTCG format, stripped frontmatter.

5 minutes to start

Works with any React component library.

1

Load your project

Point Saddle at your repo. It auto-detects component directories and parses design.md frontmatter.

2

Connect dev server

Auto-detects localhost:3000/5173/8080. Your running app appears inside Saddle's preview.

3

Edit with tokens

Figma-style inspector with token pickers. Changes save to disk. Your app updates via HMR.

4

Connect Claude Code

One-click MCP setup. Claude reads schemas, updates tokens, creates new variants.
